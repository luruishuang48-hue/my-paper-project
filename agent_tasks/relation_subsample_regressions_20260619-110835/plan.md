# Multi-agent execution plan

任务：按事件-公司关系分类分别回归，输出可用于论文正文或附录的关系机制结果。

时间：20260619-110835，Asia/Shanghai。

## 目标

1. 读取当前 clean panel 和关系面板，确认关系变量口径与样本规模。
2. 构造两类关系样本。一类为非互斥旗标样本，另一类为互斥主关系样本。
3. 对每类关系分别估计能力定价、开源调节、成本速度机制和媒体情感机制。
4. 输出 CSV、Markdown 表、系数图和简短解释报告。
5. 审阅样本量、小聚类、重叠关系和显著性解释风险，修订报告表述。

## 阶段一 信息收集与方案规划

主代理负责读取数据字段、现有关系结果和 proposal 对关系的要求。  
子代理 A 负责核查关系变量定义与样本规模，输出 `relationship_data_audit.md`。  
子代理 B 负责审阅回归设定和表格呈现，输出 `relationship_regression_review.md`。

## 阶段二 执行与整合

主代理编写并运行 `run_relation_subsample_regressions.R`。  
输出位置为 `outputs/`。  
核心结果包括关系子样本回归、互斥关系组回归、跨关系系数图和简明结果表。

## 阶段三 审阅

检查每个关系组的事件数、观测数和公司数。  
对 owner、investor、cloud 等小样本关系使用保守解释。  
检查非互斥样本和互斥样本是否给出一致方向。

## 阶段四 修订与交付

根据审阅意见修订脚本和报告。  
最终交付 `final_report.md`，说明哪些关系组显示更强能力定价，哪些关系组样本太小或结果不稳。

## 约束

不覆盖既有项目结果。  
所有新增文件只写入本任务目录。  
中文说明减少冒号使用，结果解释不夸大因果含义。
