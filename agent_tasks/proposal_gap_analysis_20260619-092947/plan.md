# Multi-agent execution plan

任务时间 20260619-092947，Asia/Shanghai

任务目标

通读项目，根据 `proposal.md` 找出研究承诺中尚未完成的分析，并在现有数据和代码允许的范围内补做一次，输出可复核的结果和说明。

阶段一 规划阶段

- 读取项目结构、`proposal.md`、已有代码、结果目录和数据字典。
- 判断 proposal 承诺的核心分析、异质性、机制、稳健性和可视化。
- 并行安排子代理梳理文档与代码结果，主代理保留关键路径，直接检查数据与可执行脚本。

阶段二 实施阶段

- 将缺失分析转成可运行任务。
- 在不破坏已有结果的前提下补充脚本或复用现有脚本。
- 生成表格、图形和日志，保存到项目结果目录或本次任务目录。

阶段三 审阅阶段

- 检查结果是否对应 proposal 的研究设计。
- 检查样本量、变量、模型设定、输出文件和可重复运行性。
- 标记无法完成或数据不足的分析。

阶段四 修订阶段

- 根据审阅结果修正脚本、表格或说明。
- 形成最终交付，列出已补做分析、文件位置、主要发现和剩余限制。

子代理计划

- 文档代理读取 proposal 和正文草稿，输出研究承诺清单到 `agent_tasks/proposal_gap_analysis_20260619-092947/doc_gap_report.md`。
- 代码代理读取脚本和结果目录，输出已有分析清单到 `agent_tasks/proposal_gap_analysis_20260619-092947/code_result_report.md`。
- 主代理整合两份报告，执行缺失分析并复核。
