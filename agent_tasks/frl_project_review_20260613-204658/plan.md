# Multi-Agent Execution Plan

任务时间 20260613-204658，北京时间。

任务目标

通读项目材料，按 Finance Research Letters 的投稿预期，判断当前论文的主要问题，并给出下一步推进方向。

执行原则

- 写作诊断以读者第一、单一贡献、可识别结果、短文结构为准。
- 输出聚焦两个问题。现在的问题。下一步的前进方向。
- 遵守项目写作要求，减少冒号使用。
- 主代理保留整合判断和最终建议，子代理只做边界清晰的并行阅读。

阶段一 规划阶段

- 快速盘点项目结构，识别论文主文、数据、代码、表格、图形和补充材料。
- 建立阅读清单，确定哪些文件必须细读，哪些文件只需抽查。
- 并行启动子代理。
  - 子代理 A 阅读主文和摘要引言，输出问题诊断到 `agent_tasks/frl_project_review_20260613-204658/main_text_review.md`。
  - 子代理 B 阅读数据、代码、表格和结果文件，输出实证可信度诊断到 `agent_tasks/frl_project_review_20260613-204658/empirical_review.md`。
  - 子代理 C 检索 Finance Research Letters 的当前投稿定位和格式要求，输出期刊匹配诊断到 `agent_tasks/frl_project_review_20260613-204658/frl_fit_review.md`。

阶段二 实施阶段

- 主代理同步阅读项目索引、核心 LaTeX 或 Markdown 文稿、结果表和代码入口。
- 整合子代理报告，识别共识问题和冲突判断。
- 将问题按投稿前阻断程度排序。

阶段三 审阅阶段

- 以 FRL 短文标准复核诊断是否过宽或过窄。
- 检查建议是否能转化为具体行动，而不是泛泛修改意见。

阶段四 修订阶段

- 形成最终中文建议。
- 输出结构为两部分。现在的问题。下一步的前进方向。
- 必要时附核心文件路径，方便用户继续修改。
