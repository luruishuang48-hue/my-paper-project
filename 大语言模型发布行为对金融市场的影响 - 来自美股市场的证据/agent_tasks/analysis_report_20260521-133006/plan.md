# Multi-Agent Execution Plan

任务：根据 `proposal.md` 分析 `事件集数据.csv`，生成完整分析报告。

时间戳：20260521-133006（Asia/Shanghai）

## 1. 规划阶段：信息收集与方案规划

- 主代理读取 `proposal.md`、识别研究问题、变量设定、模型框架和报告结构要求。
- 主代理审计 `事件集数据.csv` 的编码、列名、样本量、缺失值、变量类型和可用统计口径。
- 子代理 A：提炼 proposal 的理论框架、假设、识别策略和报告大纲，输出 `proposal_brief.md`。
- 子代理 B：独立完成数据质量检查、变量字典、异常值与缺失值诊断，输出 `data_audit.md` 和必要的表格。
- 子代理 C：根据 proposal 与数据字段设计可执行的实证分析方案，输出 `analysis_design.md`。

## 2. 实施阶段：执行与整合任务结果

- 主代理保留关键路径：读取数据结构、执行核心统计分析、生成结果表与图。
- 子代理 D：围绕事件研究、异质性、稳健性与解释口径给出独立分析建议，输出 `robustness_review.md`。
- 主代理整合各代理结果，完成描述性统计、事件分布、分组比较、相关性或回归分析，并保存中间产物。

## 3. 审阅与反馈阶段

- 子代理 E：对初稿报告做技术审阅，检查结论是否由数据支持、方法是否与 proposal 一致、是否存在过度解释，输出 `report_review.md`。
- 主代理检查中文表达、数值一致性、表图引用和局限性说明。

## 4. 修订与交付阶段

- 主代理根据审阅意见全面修订报告。
- 输出完整中文分析报告到 `agent_tasks/analysis_report_20260521-133006/final_report.md`。
- 如分析生成表格或图形，同步保存在任务目录中，并在最终回复中说明路径。
