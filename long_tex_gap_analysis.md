## 二、引言部分的逐层差距

### 范文引言的结构拆解（约2.5页）

范文引言包含**7个功能层**，每一层都不可省略：

1. **现象引入/故事钩子**（约200字）：用一个具体的、读者能共鸣的现实现象开场（鸵鸟效应——投资者在市场下跌时回避查看账户），建立"这个问题有意思且重要"的第一印象。
2. **文献回顾与研究空白**（约800字）：密集引用现有研究（20+篇），按"已经知道什么→还不知道什么→本文填补什么"的逻辑三段论展开。**每一句断言都有括号引用支撑。**
3. **本文做什么（研究问题精确界定）**（约200字）：一句话说清楚研究问题、用什么数据、看什么效应。
4. **方法论创新说明**（约300字）：交代本文方法论上的亮点（如独特的数据源、指标构造方式），解释为什么这种方法比前人更好。
5. **核心发现预览**（约300字）：用具体数字预览3-4个关键发现，每个发现一句话。
6. **边际贡献陈述**（约400字）：用"第一/第二/第三"的递进结构，明确写出3-4条贡献，每条都要对标一条具体的已有文献说明"本文相对于XXX的增量在于……"。
7. **结构安排**（约100字）："本文其余部分安排如下……"

### 我的草稿引言的问题

| 功能层 | 范文 | 我的草稿 | 诊断 |
|--------|------|----------|------|
| 现象钩子 | 有（鸵鸟效应的日常直觉） | 有（DeepSeek暴跌、NVIDIA市值蒸发） | ✓ 基本ok |
| 文献回顾 | 约800字，20+引用 | 约100字，0引用 | ❌ **致命缺陷** |
| 研究空白 | 明确指出"现有文献未能……" | 有但太笼统 | ⚠ 需更具体 |
| 方法论亮点 | 独立段落说明 | 混在研究策略段里 | ⚠ 不够突出 |
| 核心发现预览 | 3-4句，带具体数字 | 有（3点） | ✓ 基本ok |
| 边际贡献 | 3条，每条对标文献 | 有3条但未对标任何文献 | ❌ **缺对标** |
| 结构安排 | "本文其余部分安排如下" | 有 | ✓ |

**核心问题：引言中无任何文献引用。《金融研究》的引言是"引用驱动"的——每一个判断背后都有文献支撑。我的引言读起来像观点陈述，不像学术论文。**

## 三、文献综述与理论分析的致命差距

### 范文的做法

范文的"二、理论分析与研究假设"包含：

1. **正式的数理模型**（约1.5页）：
   - 建立投资者效用函数（CARA效用、信息不对称）
   - 引入"选择性关注"的形式化定义
   - 通过均衡求解推导出可检验的命题（Proposition）
   - 命题直接对应后文的研究假设

2. **从命题到假设的推导**（约0.5页）：
   - 假设1、假设2、假设3分别对应模型的不同推论
   - 每个假设不仅说"预测A成立"，还说"之所以预测A成立，是因为模型中的XX机制"

3. **文献密度**：
   - 理论部分引用15-20篇论文
   - 每一步逻辑推理都标注"这与XXX（2019）的框架一致"或"本文区别于YYY（2020）的假设在于……"

### 我的草稿的问题

| 方面 | 范文 | 我的草稿 | 诊断 |
|------|------|----------|------|
| 数理模型 | 有完整的效用最大化模型 | 无 | ❌ **最大差距** |
| 文献综述深度 | 3条文献线，每条5-10篇详细讨论 | 3条文献线，每条约200字、0篇引用 | ❌ **空壳** |
| 假设推导 | 从模型命题逻辑推导 | 从直觉/常识推导 | ❌ 缺理论严谨性 |
| 假设数量 | 3-4个，有明确的方向预测 | 3个，方向预测有 | ✓ 数量ok |
| 文献对话 | "区别于XXX，本文……" | 无 | ❌ |

**核心问题：中文顶刊要求假设来自"理论推导"而非"直觉断言"。即便不写完整的数理模型，至少需要一个框架性的理论论证（如产业组织的互补品/替代品模型的形式化表述），从框架推出可检验的命题，再映射为实证假设。我的草稿跳过了这一步，直接从"经济直觉"跳到"假设"。**

## 四、研究设计部分的差距

### 范文的做法

范文的"三、研究设计"包含：

1. **样本选取**：非常详细地说明数据来源（CSMAR/WIND等）、时间范围、筛选标准（剔除ST、金融行业等）、最终样本量。

## 六、稳健性检验的差距

### 范文的稳健性检验（约2.5页，6-8项）

范文通常包含以下类型的检验：

1. 替换被解释变量的度量方式
2. 替换核心解释变量的度量方式
3. 子样本分析（按时间、行业、规模等分组）
4. **内生性处理**（PSM-DID、工具变量、Heckman两步法）——这是最重要的
5. 遗漏变量偏误讨论
6. 安慰剂检验（Placebo test）
7. 更换估计方法

### 我的草稿的问题

| 检验类型 | 范文 | 我的草稿 | 诊断 |
|----------|------|----------|------|
| 替换变量 | 有 | 有（控制AA Intelligence） | ✓ |
| 子样本 | 有 | 有（剔除DeepSeek） | ✓ |
| Leave-one-out | 不常见但OK | 有 | ✓ |
| **内生性/因果识别** | **必备**（PSM/IV/DID） | **完全缺失** | ❌ **致命** |
| 安慰剂检验 | 常见 | 缺失 | ❌ 应补 |
| Bootstrap | 不常见 | 有 | ✓ |
| **替换CAR窗口** | 常见 | 有（R2全窗口） | ✓ |

**核心问题：中文顶刊审稿人最关注内生性。本文的识别策略（事件研究）天然比DID弱——事件不是随机分配的，公司的"位置"也不是随机分配的。必须至少讨论（最好正式处理）以下内生性来源：**

1. **遗漏变量**：位置可能与其他公司特征相关（规模、行业、机构持股等），这些特征本身影响对AI事件的反应。
2. **反向因果**：不太可能，但要说明。
3. **自选择**：公司进入某个位置不是随机的。可用PSM或Entropy Balancing匹配上下游公司的其他特征。

## 七、进一步分析/机制检验的差距

### 范文的做法

范文的"进一步研究"通常包含：

1. **机制检验（Mechanism/Channel Tests）**：不是简单讨论，而是用中介变量或分组回归来"验证"机制。例如：如果效应通过"信息不对称"渠道传导，那么信息不对称更严重的子样本效应应该更大。
2. **异质性分析**：按公司特征（规模、分析师覆盖、机构持股）分组。
3. **经济后果**：效应对实体经济有什么意义（投资、融资等）。

### 我的草稿

我的"六、进一步分析"是纯**文字讨论**（为什么只有R4显著、开闭源机制），没有**新的回归/分组检验**来支撑。中文顶刊审稿人会觉得这是"凭空议论"而非"实证验证机制"。

**需要补充的**：
- 按公司特征分组（如：公司规模大小、分析师覆盖度高低、AI业务收入占比高低）看下游效应是否有差异
- 机制变量中介检验（如：若通过"竞争威胁"渠道，则行业集中度低的子样本效应更强）
